# Agent Teams 결과 취합 규칙

Agent Teams 3명의 결과를 받은 후 다음 절차로 취합할 것.

## Step 0: 인용 검증
각 에이전트의 원문 인용이 실제 문서에 존재하는지 확인.
존재하지 않는 인용이 포함된 이슈는 확신도를 1로 강제 하향.

## Step 1: Reject Filter
확신도 < 5인 이슈는 투표에서 제외.
제외된 이슈는 마지막에 '📎 Low Confidence (참고용)' 섹션에 표시.

## Step 2: 이슈 매칭
같은 문제를 지적한 이슈끼리 묶기.
기준: 동일 위치의 동일 항목을 지적하거나, 근본 원인이 같은 경우.
비슷해 보이지만 근본 원인이 다른 이슈를 억지로 묶지 말 것.

## Step 3: 가중 판정
묶인 이슈 그룹마다 판정:

- 🔴 High Confidence: 유효(확신도≥5) 2명 이상 지적
- 🟡 Needs Review:
  - 유효 1명 지적, 확신도 8+ AND 근거강도 4+
- ⚪ Low Priority: 유효 1명, 확신도 5-7

가중 점수 = Σ(확신도 × 근거강도) / 3
가중 점수를 임의로 조정하거나 반올림하지 말 것. 공식 그대로 계산할 것.

판정 예시:
- Agent A만 ISS-003 지적 (확신도=9, 근거강도=4) → 🟡 Needs Review
- Agent B만 ISS-007 지적 (확신도=6, 근거강도=2) → ⚪ Low Priority
- Agent A,B가 ISS-001 지적 → 🔴 High Confidence

## Step 4: 불일치 분석
의견이 갈린 이슈(일부는 CRITICAL, 일부는 미발견):
- 왜 갈렸는지 분석
- 놓친 건지 과잉 반응인지 판단
- 승격 또는 강등 추천 + 근거

## Step 5: 오류 패턴 분석
- 특정 위치에 이슈 집중 여부
- 특정 유형의 불일치가 함께 나타나는 패턴 여부
- 패턴이 있으면 해당 영역의 전반적 품질 문제 지적

## 취합 결과 저장 형식

```
### 📊 취합 요약
| # | 이슈 | 유형 | 판정 | 지적 수 | 가중점수 | 심각도 |
|---|------|------|------|---------|----------|--------|

### 🔴 High Confidence 이슈
(각 이슈: 추론 과정 + 검증된 원문 인용 + 제안)

### 🟡 Needs Review 이슈
(각 이슈 상세 + 승격/강등 추천이 있으면 포함)

### 🔍 불일치 분석
(갈린 이슈별 분석 + 승격/강등 추천)

### 📈 오류 패턴 분석
(집중도 + 유형 공출현 패턴 + 전반적 품질 진단)

### ⚪ Low Priority 이슈
(상세)

### 📎 Low Confidence (참고용)
(확신도 < 5 또는 인용 검증 실패로 제외된 이슈 목록)
```

## 루프 판정 기준

다음 조건 중 하나라도 해당하면 문서 수정 후 다음 라운드로:
- 🔴 High Confidence 이슈 중 CRITICAL 또는 MAJOR 심각도가 1개 이상 존재
- 🟡 Needs Review 이슈 중 CRITICAL 심각도가 1개 이상 존재

위 조건 모두 해당 없으면 다음 Phase로 이동.

추가 종료 조건:
- 최대 5라운드 도달 시 남은 이슈와 함께 다음 Phase로 이동
- 직전 라운드와 동일한 이슈가 반복되면 (수정이 효과 없으면) 다음 Phase로 이동

## 문서 수정 규칙

🔴 High Confidence 이슈는 문서에 바로 반영할 것.
🟡 Needs Review 이슈는 **반영하지 않고 기록만** 해둔다. Phase 5에서 자율 판단하여 반영한다. 중간 Phase에서 사용자에게 묻고 멈추지 말 것.

수정 시:
1. 🔴 High Confidence의 CRITICAL/MAJOR 이슈를 **모두** 반영
2. 🟡 Needs Review는 반영하지 않고 리뷰 결과 파일에 기록만 한다
3. 수정 시 새로운 모순을 만들지 않도록 관련 영역을 교차 확인

이슈가 전혀 없으면 '전원 이슈 없음 보고'로 끝낼 것. 빈 테이블을 채우려고 억지 이슈를 만들지 말 것.
